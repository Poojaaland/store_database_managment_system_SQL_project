# store_database_managment_system_SQL_project

This project is based on the concept of managing and analyzing data for a retail store using SQL. It is divided into two parts, each focusing on diffrent aspects of database creation and query execution.

Part 1 - Database Design & Creation

1. Designed the ER diagram and created the database structure accordingly.
2. Created 9 tables(in CSV format) representing key store entities such as customers, brands, categories, order_items, orders, products, staffs, stocks, stores.
3. Defined appropriate primary keys, forigen keys, and cnstraints to maintain data integrity.
4. Imported all CSV data into SQL tables.

Part 2 - SQL Queries & Analysis 

1. Performed various SQL operations and analytical queries based on the created database.
   
2. Queries include
             i.   Data retrieval various SQL join, group by, and having clauses.
             ii.  Subqueries for detailed data insights.
             iii. Ranking and aggreation using window fuctions.
             iv.  Generating sales and staff performance reports.

Tools & Technoogies Used
     1. MySQL / SQL Server
     2. ER Diagram tools
     3. CSV files for data input

About :-

This project demonstrates a practical approach to database creation, data managment, and performing analytical queries - useful for understanding real-world data handling in retail management.
